## Area of focus
1. Users should be able to start group chats with multiple participants
2. Users should be able to send/receive messages
3. Users should be able to receive messages sent while they are not online
4. Users should be able to send/receive media in their messages
5. Blue tick/Seen message feature
6. Online / Offline indicator - How can we handle a "last seen" functionality?
7. How can we handle billions of simultaneous users?
8. What do we do to handle multiple clients for a given user? App/webapp
9. What happens if the WebSocket connection fails?
10. What happens if Redis fails to deliver a message?
11. How do we handle out-of-order messages?
